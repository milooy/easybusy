# 세션 마무리 - 학습 내용 정리

이번 세션의 대화 내역을 돌아보고, 다음 기준으로 `.claude/rules/`에 기록한다.

## 기준

기록할 만한 내용:
- 이 프로젝트에서 반복적으로 쓰일 패턴이나 규칙
- 처음 알게 된 도메인 지식 (구조, 계산 방식, 데이터 흐름 등)
- 실수하거나 헷갈렸던 부분의 해결책

기록하지 않을 내용:
- 이번 한 번만 쓸 구현 세부사항
- 이미 CLAUDE.md에 있는 내용

## 절차

1. 세션 대화를 돌아보며 기록할 인사이트를 추출한다.
2. 주제별로 묶어 적절한 파일명을 결정한다. (예: `code-style.md`, `auth-flow.md`)
3. 기존 파일이 있으면 병합, 없으면 신규 생성한다.
4. 각 파일은 **핵심만** 담는다. 예시 코드는 꼭 필요한 경우만.
5. 변경된 파일을 커밋한다.

## 출력 형식

작업 완료 후 다음을 보여준다:
- 생성/수정된 파일 목록과 각각 어떤 내용을 담았는지 한 줄 요약
